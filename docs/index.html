How to Use This Tool

Overview

This tool generates AI prompt templates for Salesforce development using prompt engineering best practices. It structures prompts with Role, Context, Constraints, Guardrails, and Outcomes to help AI tools produce better, more accurate results.

Step-by-Step Guide

  1. Persona: Select who is asking (Developer, Architect, or Business Analyst). This sets the expertise level and perspective.
  2. Artifact Type: Select one or more types of Salesforce components you're building (LWC, Apex, Test Classes, Flows, or Objects).
  3. Work Product: Choose what you want to generate - Build (implementation), Story (user story/acceptance criteria), or Design (architecture/technical design).
  4. Org Mode: Select Greenfield (build from scratch) or Existing Org (analyze existing components first).
  5. Goal: Describe what should be delivered. Be specific about the functionality you need.
  6. Primary Objects: List the Salesforce objects involved (e.g., Case, Account, CustomObject__c).
  7. Users/Personas: Identify who will use this (e.g., Support Agent, Manager).
  8. Requirements: List functional and non-functional requirements, edge cases, and any special considerations.
  9. Constraints: Standard constraints are automatically included. Add any additional constraints specific to your project.
  10. Advanced (optional): Add org details, integration information, or other context that might be relevant.

Understanding the Output

The generated prompt includes:

  • Role: Defines the AI's persona and expertise level
  • Context: Provides all the relevant information about your project
  • Constraints: Lists limitations and requirements that must be followed
  • Guardrails: Safety and best practice rules to prevent common mistakes
  • Outcomes: Defines what "done" looks like for your work product
  • Process: Steps the AI should follow (discovery for existing orgs, assumptions for greenfield)
  • Output Format: Specifies how the response should be structured

Standard Guardrails & Constraints

The following guardrails and constraints are automatically included in every generated prompt:

Standard Constraints (Always Applied)

  • No hardcoded record IDs, profile IDs, or endpoint URLs. Use metadata, Custom Metadata/Settings, Named Credentials, and labels where appropriate.
  • Be governor-limit aware and bulk-safe (especially for Apex and record-triggered automation).
  • Follow Salesforce best practices and the org's established patterns and naming conventions.
  • Prefer secure-by-default design: least privilege, CRUD/FLS, sharing, input validation, and safe error messages.

Base Guardrails

  • Do NOT invent org-specific names/IDs. If missing, ask questions or state assumptions explicitly.
  • If requirements conflict, call out the conflict and propose options rather than guessing.
  • If any information is missing or ambiguous, first list clarifying questions and assumptions before writing any code.
  • If you cannot safely proceed, output clarifying questions instead of code.
  • Do not explain the prompt back to me or talk about being an AI. Go straight to the engineering spec.
  • Output must be copy/paste ready and organized using clear headings and checklists.

Salesforce-Specific Guardrails

  • Explain how the solution aligns with Salesforce best practices and what trade-offs were made.

Note: Additional artifact-specific guardrails are automatically added based on the artifact types you select (LWC, Apex, Flow, etc.).

Tips for Best Results

  • Be specific in your goal and requirements - vague inputs lead to vague outputs
  • For existing orgs, list known components to help the AI understand your environment
  • Include security, performance, and compliance requirements in constraints
  • Use the "Existing Org" mode when working with established Salesforce orgs to avoid duplicating functionality
  • Review the generated prompt before using it - you can edit it directly if needed

Copying and Using the Prompt

Once generated, click "Copy prompt" to copy the entire prompt to your clipboard, then paste it into your AI tool (ChatGPT, Claude, Cursor, etc.). The prompt is designed to guide the AI to ask clarifying questions if information is missing rather than making assumptions.
